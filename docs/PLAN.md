# План развития LocalAI RAG

## Этап 1. Локальный notebook по папкам

Готово в текущем MVP:

- UI для добавления папки по пути.
- Кнопка индексации выбранного источника.
- Локальный Markdown-кэш на диске `C:`.
- Manifest неизмененных файлов.
- Keyword-поиск по chunk-ам.
- Базовый chat endpoint: без LLM возвращает найденные источники, с OpenAI-compatible endpoint использует контекст.

## Этап 2. Надежная обработка документов

- Подключить Docling для PDF с таблицами, layout и OCR.
- Добавить очередь индексации и отмену задач.
- Показывать ошибки по файлам в UI.
- Добавить режим "обновить индекс" без полного пересчета.

## Этап 3. Семантический RAG

- Поднять Qdrant из `compose.yaml`.
- Добавить embeddings для chunk-ов.
- Добавить hybrid retrieval: keyword + vector.
- Добавить reranker для точности.
- Выводить источники с путем, страницей и фрагментом.

## Этап 4. NotebookLM-поведение

- Несколько notebooks/коллекций.
- Чат по одному notebook или по нескольким выбранным папкам.
- Сводки документов.
- История вопросов.
- Избранные источники.
