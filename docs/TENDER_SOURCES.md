# Тендеры и договоры

В LOCAL_RAG теперь два типа источников:

| Тип | `sourceType` | Где лежат | Назначение |
|-----|--------------|-----------|------------|
| **Договор** | `contract` (по умолчанию) | `\\192.168.5.55\SharA_Tender\...` | Выигранные тендеры, договоры |
| **Тендер** | `tender` | `G:\Мой диск\003 Тендеры 2025\...` | Материалы в процессе и после тендера |

Тендеры **не подмешиваются** в список проектов чата. В чате выбираются только договоры.

Когда тендер привязан к договору (`linkedContractId`), поиск и чат по договору автоматически включают файлы тендера.

## Синхронизация тендеров с G:\

```powershell
# План: какие папки станут tender-источниками
npm run agent:sync-tenders

# Записать в config/sources.yaml
npm run agent:sync-tenders:apply

# Индексация
npm run agent:run
```

То же самое доступно из UI: **Настройки → Источники → Синхронизировать тендеры с G:\\**.
Первый запуск показывает dry-run отчёт, кнопка в отчёте применяет запись в `config/sources.yaml`.

Скрипт `scripts/sync-tender-sources.mjs`:

- сканирует `G:\Мой диск\003 Тендеры 2025` (категории: В работе, Завершенные, Проиграли…);
- создаёт отдельный source на каждую папку тендера;
- **не трогает** существующие договоры;
- для категории **Завершенные** пытается автоматически привязать тендер к договору по названию;
- для **В работе** и **Проиграли** — только индексирует как отдельные тендеры без автопривязки.

Переменные окружения:

```text
RAG_TENDER_ROOT=G:\Мой диск\003 Тендеры 2025
RAG_TENDER_CATEGORIES=В работе|Завершенные|Проиграли, отказ в участии
RAG_TENDER_AUTO_LINK_CATEGORIES=Завершенные
```

## Ручная привязка тендера к договору

В UI выберите tender-источник в **Настройки → Источники**, укажите договор в выпадающем списке и нажмите **Сохранить**.
API для той же операции:

```http
PUT /api/sources/tender-6821a03a4d
Content-Type: application/json

{ "linkedContractId": "source-7c203d0b4a" }
```

В `config/sources.yaml` это хранится у tender-источника:

```yaml
- id: tender-6821a03a4d
  title: 272. Садовническая 76 (Балчуг Эстейт)
  path: G:\Мой диск\003 Тендеры 2025\Завершенные\272. ...
  sourceType: tender
  tenderCategory: Завершенные
  linkedContractId: source-7c203d0b4a
```

После выигрыша тендера: создайте/обновите договор на шаре и укажите `linkedContractId`.

## Ручные mappings для спорных пар

Если имя папки неоднозначное, создайте локальный `config/tender-links.yaml` по примеру
`config/tender-links.example.yaml`:

```yaml
mappings:
  - tenderFolder: "256. Primavera K14 (Spartak)"
    contractId: source-example-contract
```

`sync-tender-sources` читает эти mappings до автоматического match и применяет их с приоритетом.
Файл может содержать `tenderFolder`, `tenderPath` или `tenderId`; в репозиторий добавлен только example без реальных путей.

## Google Sheets (.gsheet)

Ярлыки `.gsheet` в папке G:\ не индексируются как файлы. Для таблиц используйте context links или discovery-агент (`docs/GOOGLE_DISCOVERY_AGENT.md`).
